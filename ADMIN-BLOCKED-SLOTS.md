# 鎖定時段（不開放預約）操作說明

給負責施法/放 Buff 的人在**管理頻道**（`MANAGEMENT_CHANNEL_ID`）使用，不需要碰 VM 或資料庫。
這個頻道跟機器人平常推播紀錄用的 `ADMIN_CHANNEL_ID` 是分開的兩個頻道。

## 鎖定某個時段

在管理頻道貼：

```
鎖定：
日期：07/20
開始：20:00
結束：21:00
原因：離線維護
```

- `日期` 格式 `MM/DD`，必填
- `開始`、`結束` 格式 `HH:MM`（24 小時制），必填，開始要早於或等於結束
- `原因` 選填，會顯示在公告跟擋下的提示訊息裡

送出後機器人會自動：
1. 把這個時段設為不開放預約（之後任何人在這個時段留言預約都會被擋下並附上原因）
2. 找出這個時段內**已經存在**的預約，直接從班表刪除
3. 在獨立的**公告頻道**（`ANNOUNCEMENT_CHANNEL_ID`）發 `@everyone` 公告，說明時段不開放，並 tag 出所有被取消預約的人（不會混進討論串裡）
4. 回覆你（在管理頻道）鎖定結果，包含這次鎖定的**編號**與取消了幾筆預約

**記得記下回覆裡的編號**，之後要解除鎖定需要用到。

## 解除鎖定

```
解除鎖定：
編號：7
```

`編號` 換成鎖定時機器人回覆給你的那個數字。解除後機器人會：
1. 移除這個鎖定設定（之後這個時段恢復可以預約）
2. 在公告頻道發一則 `@everyone` 的「恢復開放」公告
3. 回覆你解除結果

## 復原被取消的預約

鎖定時段時被系統自動取消的預約，是「軟刪除」，資料還在，可以復原。

先查出要復原的 id（狀態是 `cancelled` 的才是被取消的）：

```bash
sqlite3 bookings.db "SELECT id, location, scheduled_time, channel, booker_id, status FROM bookings WHERE booking_date='2026-07-20' ORDER BY scheduled_time;"
```

再執行：

```bash
node scripts/admin-manage-booking.js restore 5
```

會自動把狀態改回 `confirmed`，重新出現在班表上，並同步更新 embed。

⚠️ 復原不會自動檢查有沒有跟其他預約衝突（例如同時段又被別人訂走了），復原前建議先確認一下該時段目前的狀況。

## 要異動時段（例如時間改一下）

沒有直接的「修改」指令，請照下面兩步：
1. 先 `解除鎖定` 舊的
2. 再重新下一次 `鎖定` 指令，填新的時間

## 注意事項

- 這兩個指令**只在管理頻道（`MANAGEMENT_CHANNEL_ID`）有效**，在討論串或機器人紀錄頻道貼一樣的文字不會被當成指令
- 判斷是不是指令，只看留言的**第一行**是不是「鎖定」或「解除鎖定」開頭，其他一般聊天/客服對話完全不受影響
- 目前沒有做「同一時段被鎖定兩次」的防呆，如果不小心鎖定兩次相同時間，會有兩筆重複的鎖定紀錄，功能上不影響（都會擋下預約），但看起來會有點亂，建議鎖定前先確認沒有重複下過指令
